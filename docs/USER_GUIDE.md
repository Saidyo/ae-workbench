# AE Workbench 说明文档

更新时间：2026-06-27

## 1. 系统定位

AE Workbench 是一个本地桌面素材与工程管理工具，面向视频设计师、AE 动效师和本地素材量较大的创作者。

系统的核心目标是把本地 AE 项目、人物立绘、图片素材、视频素材、参考资料、AE 工程文件和 Eagle 素材库统一索引到一个可视化工作台中，方便按项目查看、筛选、预览和打开原始文件。

这个系统默认采用“原位链接”的管理方式：素材仍保留在原本的本地目录中，系统只记录路径、分类、标签、时间、项目关联和预览信息，避免因为复制素材造成重复占用硬盘空间。

## 2. 适用场景

- 管理多个 AE 项目及其工程目录。
- 按项目查看该项目实际使用或关联过的素材。
- 管理人物立绘、参考图、参考视频、AE 工程、PSD、模板、音频等素材。
- 将本地文件夹中的新增素材自动同步到系统索引。
- 将 Eagle 素材库中的文件夹、标签、评分、备注等信息单向同步到系统。
- 快速预览图片和视频素材，并打开文件或定位到文件夹。
- 查看每日新增素材数量和入库趋势。

## 3. 技术形态

系统采用 Electron + Vite + React + TypeScript 构建。

| 层级 | 说明 |
| --- | --- |
| 桌面壳 | Electron，负责本地文件访问、文件夹监听、系统弹窗和打开文件 |
| 前端界面 | React + TypeScript，负责项目库、素材库、统计和设置页面 |
| 构建工具 | Vite |
| 本地索引 | `data/app-data.json` |
| 文件监听 | `chokidar` |
| Eagle 联动 | Eagle 本地 API + `.library/images/*.info/metadata.json` 只读扫描 |

## 4. 启动方式

### 4.0 给别人首次部署

把整个项目文件夹交给对方后，推荐让对方双击：

```text
一键部署并打开AE Workbench.cmd
```

这个脚本会完成以下事情：

- 检查电脑里是否有 Node.js 和 npm。
- 创建 `Library/`、`Projects/`、`Cache/`、`data/` 目录。
- 安装项目依赖。
- 构建 Electron 桌面版。
- 检测 Eagle 本地 API。
- 打开 AE Workbench 桌面窗口。

如果提示缺少 Node.js，请先安装 Node.js LTS，然后重新双击部署脚本。

推荐双击当前目录下的一键脚本：

```text
open-ae-workbench.cmd
```

也可以双击中文快捷脚本：

```text
一键打开AE Workbench.cmd
```

开发模式启动：

```bash
npm install
npm run dev
```

生产构建：

```bash
npm run build
```

类型检查：

```bash
npm run typecheck
```

注意：图片、视频预览、本地文件打开、文件夹选择、Eagle 同步等能力需要通过 Electron 桌面版打开。如果只在浏览器里访问 `http://127.0.0.1:5173/`，部分本地文件能力会不可用。

## 4.1 安全与性能说明

- 桌面版通过 Electron preload 暴露固定 API，前端页面不能直接访问 Node.js 能力。
- 本地文件预览走 `asset://local/...` 协议，只服务系统索引中已登记的素材或缩略图路径。
- 素材库搜索会延迟执行筛选，适合大量素材时连续输入关键词。
- 视频卡片默认只读取基础 metadata，鼠标悬停时再播放预览，减少素材库初次打开压力。
- 推荐在较大改动后运行 `npm run typecheck` 和 `npm run build`，再检查桌面与窄屏布局。

## 5. 本地目录结构

系统会在项目根目录下维护以下目录：

```text
Library/       素材库目录
Projects/      AE 项目目录
Cache/         缓存目录
data/          本地索引数据
```

### 5.1 Library 素材类型目录

系统会按素材类型维护素材库目录：

```text
Library/
  Characters/
  Images/
  Videos/
  Audio/
  References/
  AE/
  PSD/
  Templates/
  Misc/
```

当前系统以链接索引为主。导入本地素材或文件夹时，默认记录原路径，不复制文件。

### 5.2 Projects 项目目录

创建项目时，系统会自动生成标准 AE 项目目录：

```text
Projects/
  项目名/
    01_AEP/
    02_Footage/
    03_Images/
    04_Audio/
    05_References/
    06_Renders/
    07_Delivery/
    99_Archive/
```

项目目录用于承载 AE 工程和项目上下文。系统会记录项目创建时间、修改时间、最近打开时间和项目状态。

## 6. 主要功能模块

### 6.1 概览

概览页用于快速查看系统当前状态，包括：

- 项目数量。
- 素材总量。
- 今日新增素材。
- 本地服务状态。
- 当前选中项目。
- 最近素材和统计概况。

### 6.2 项目库

项目库用于创建、切换和管理 AE 项目。

支持能力：

- 创建项目。
- 自动生成标准项目目录。
- 查看项目状态：进行中、暂停、完成、归档。
- 打开项目文件夹。
- 记录创建时间、修改时间和最近打开时间。
- 在当前项目下查看关联素材。
- 将本地素材或文件夹关联到当前项目。
- 项目素材支持缩放浏览。

项目库中的素材和素材库中的项目分类是同步的。选择某个项目后，可以直观看到该项目下关联的素材，而不是把所有项目素材混在一个目录里。

### 6.3 素材库

素材库是系统的核心浏览区域。

支持能力：

- 按全部素材或指定项目查看。
- 按素材类型筛选：人物立绘、图片、视频、音频、参考、AE 工程、PSD、模板、其他。
- 按时间筛选：全部时间、今天、近 7 天、近 30 天、自定义时间。
- 按入库时间或文件修改时间排序。
- 支持正序和倒序切换。
- 支持搜索文件名、路径、Eagle 标签、Eagle 文件夹、备注等信息。
- 支持屏蔽某类素材，比如 AE 工程、音频、模板等。
- 支持屏蔽某个项目的素材。
- 支持滚动自动加载更多素材。
- 支持 Ctrl + 鼠标滚轮缩放素材卡片。
- 左侧筛选栏支持拖拽调整宽度，双击分隔线恢复默认宽度。

### 6.4 图片和视频预览

素材卡片支持内嵌预览：

- 图片：`jpg`、`jpeg`、`png`、`webp`、`gif`、`bmp`、`tif`、`tiff`、`svg`。
- 视频：`mp4`、`mov`、`mkv`、`webm`、`avi`、`m4v`。
- 音频：记录为音频素材，可打开本地文件。
- AE 工程、PSD、模板等非媒体文件会显示文件类型预览。

预览走 `asset://local/...` 本地协议，由 Electron 读取原始文件路径。视频支持 range 请求，用于浏览器视频控件正常加载和拖动。

### 6.5 每日统计

每日统计页用于查看素材入库节奏。

支持能力：

- 查看今日新增素材数量。
- 查看最近 7 天新增趋势。
- 按类型拆分统计图片、视频、音频、人物、参考、AE 工程等数量。
- 记录每日新增总量和文件总大小。

### 6.6 设置

设置页用于维护本地目录、同步状态和系统索引。

支持能力：

- 打开素材库目录。
- 打开项目库目录。
- 打开缓存目录。
- 检测 Eagle 本地 API。
- 选择 Eagle `.library` 目录。
- 立即同步 Eagle。
- 取消 Eagle 关联。
- 重新扫描本地索引。
- 清理已不存在的素材记录。
- 恢复默认浏览。
- 取消关联已监听的本地文件夹。

“恢复默认浏览”会重置素材库的浏览状态，包括：

- 查看全部素材。
- 清空搜索。
- 清空类型屏蔽。
- 清空项目屏蔽。
- 清空 Eagle 筛选。
- 清空时间筛选。
- 缩放恢复到默认值。
- 排序恢复为按入库时间倒序。

## 7. 本地素材同步逻辑

系统会监听以下位置：

```text
Library/
Projects/
已关联的外部本地文件夹
```

同步规则：

- 当这些目录中新增文件时，系统会自动登记为素材。
- 当项目目录中新增文件时，系统会自动识别所属项目并建立项目关联。
- 当被监听文件被删除时，系统会移除对应素材索引。
- 系统不会读取或登记 `.json` 文件。
- 系统会忽略 `data/`、`Cache/`、`node_modules/` 等内部目录。

文件夹关联规则：

- 通过系统关联一个外部文件夹后，该文件夹会加入监听列表。
- 文件夹内新增素材会自动出现在系统中。
- 取消关联文件夹时，本地文件不会被删除，只会移除系统里的索引和项目关联。

## 8. 项目与素材的关联方式

系统中的素材可以来自三类来源：

| 来源 | 说明 |
| --- | --- |
| 本地链接素材 | 用户选择本地文件或文件夹，系统记录原路径 |
| 项目关联素材 | 用户将本地素材关联到某个项目，系统记录项目关系，不复制文件 |
| Eagle 素材 | 从 Eagle 只读同步过来的素材，系统记录原图路径和元信息 |

项目关联不会把素材复制进项目目录。系统只记录“这个项目使用了这个素材”。这样可以避免重复占用硬盘空间，同时保留项目维度的浏览能力。

## 9. Eagle 单向同步

系统支持将 Eagle 素材库单向同步到 AE Workbench。

同步方向：

```text
Eagle -> AE Workbench
```

不会发生：

- 不会写回 Eagle。
- 不会修改 Eagle 文件。
- 不会移动 Eagle 文件。
- 不会复制 Eagle 原图。

### 9.1 同步方式

系统会优先尝试连接 Eagle 本地 API：

```text
http://127.0.0.1:41595
http://localhost:41595
```

如果 Eagle API 可用，系统会读取应用和库信息，并辅助获取文件夹信息。

为了避免 Eagle API 分页或路径缺失导致只能同步一部分素材，系统同时支持直接扫描 `.library` 目录：

```text
你的 Eagle 库.library/
  images/
    xxx.info/
      metadata.json
      原图文件
      thumbnail.png
```

系统会读取 `.info/metadata.json`，并把 Eagle 原图路径作为素材路径登记到系统中。缩略图只作为预览辅助，不会替代原图路径。

### 9.2 同步的信息

Eagle 同步会尽量带入：

- 原图路径。
- 缩略图路径。
- Eagle 文件夹。
- Eagle 标签。
- 评分。
- 备注。
- 来源 URL。
- 图片宽高。
- 视频时长。
- 文件大小。
- Eagle 创建、更新、导入时间。
- 文件可用状态。

### 9.3 Eagle 筛选

素材库支持 Eagle 专属筛选：

- 来源：全部来源、本地、Eagle。
- Eagle 文件夹。
- Eagle 标签。
- 评分：全部评分、有评分、5 星、4 星以上、3 星以上。
- 状态：全部状态、可用、文件缺失、未在 Eagle 列表。

## 10. 素材类型识别

系统根据文件扩展名识别素材类型。

| 类型 | 扩展名 |
| --- | --- |
| 图片 | `.jpg`、`.jpeg`、`.png`、`.webp`、`.gif`、`.bmp`、`.tif`、`.tiff`、`.svg` |
| 视频 | `.mp4`、`.mov`、`.mkv`、`.webm`、`.avi`、`.m4v` |
| 音频 | `.mp3`、`.wav`、`.aac`、`.flac`、`.ogg`、`.m4a` |
| AE 工程 | `.aep`、`.aepx` |
| PSD | `.psd`、`.psb` |
| 模板 | `.ffx`、`.mogrt` |
| 其他 | 未匹配到以上规则的文件 |

## 11. 本地索引数据

系统当前使用本地 JSON 文件作为索引：

```text
data/app-data.json
```

索引中保存：

- 项目信息。
- 素材信息。
- 项目与素材的关联关系。
- 每日统计。
- 外部监听文件夹。
- 被忽略的素材路径。
- Eagle 来源。
- Eagle 文件夹。
- Eagle 标签。
- Eagle 同步记录。

注意：索引文件只是系统数据库，不是素材本体。删除或移动真实素材文件，会导致系统中的对应记录变成失效或被清理。

## 12. 常用操作流程

### 12.1 创建一个 AE 项目

1. 打开系统。
2. 进入“项目库”。
3. 输入项目名称。
4. 创建项目。
5. 系统会在 `Projects/` 下生成项目目录。
6. 点击项目即可查看项目详情和关联素材。

### 12.2 关联本地素材到素材库

1. 进入“素材库”。
2. 点击关联本地素材。
3. 选择文件或文件夹。
4. 系统登记原路径，不复制文件。
5. 素材会出现在素材库中，并可按类型、时间、项目或 Eagle 信息筛选。

### 12.3 关联素材到当前项目

1. 进入“项目库”。
2. 选择一个项目。
3. 点击关联素材。
4. 选择本地文件或文件夹。
5. 系统将素材记录到当前项目下，不复制文件。
6. 切换到素材库中的该项目分类，可以只查看这个项目的素材。

### 12.4 同步 Eagle 素材库

1. 打开 Eagle。
2. 进入系统“设置”。
3. 点击“检测 Eagle”。
4. 如果检测不到或路径不完整，点击“选择库”选择 `.library` 目录。
5. 点击“立即同步”。
6. 同步完成后，在素材库中使用 Eagle 文件夹、标签、评分等条件筛选。

给别人部署时注意：

- Eagle 不会跟随本系统一起安装，必须由对方自己安装并打开。
- Eagle 素材路径是对方电脑上的本地路径，不要直接复用你电脑上的 `data/app-data.json` 作为正式数据。
- 如果对方的 Eagle API 无法访问，使用“选择库”方式选择 `.library` 目录即可。
- 同步是单向只读，不会修改 Eagle 库。

### 12.5 清理失效素材

1. 进入“设置”。
2. 点击“清理失效”。
3. 系统只移除不存在的素材记录。
4. 本地文件不会被删除。

## 13. 注意事项

- 推荐始终通过 `open-ae-workbench.cmd` 打开桌面版。
- 系统以“链接原文件”为主，不建议把同一批素材重复复制到多个目录。
- `.json` 文件不会被登记为素材。
- Eagle 同步是只读同步，不会写回 Eagle。
- 取消关联素材或文件夹只删除系统索引，不删除本地文件。
- 如果预览不可用，优先确认是否使用 Electron 桌面版打开。
- 如果移动了本地素材目录，需要重新关联或重新扫描。

## 14. 当前版本边界

当前系统已经具备本地可用的项目与素材管理能力，但仍属于本地工作台版本。

已实现：

- 本地项目管理。
- 本地素材链接索引。
- 文件夹监听自动同步。
- 图片和视频预览。
- 项目维度素材分类。
- 时间筛选和排序。
- 类型屏蔽和项目屏蔽。
- 每日上传统计。
- Eagle 单向同步。
- 设置页维护工具。

暂未作为核心目标：

- 多用户协作。
- 云端同步。
- 权限管理。
- 在线分享。
- 写回 Eagle。
- 解析 AE 工程内部图层和合成结构。
- 自动生成视频缩略图或代理文件。
